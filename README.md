# Архив документтерин көзөмөлдөө системасы

Архив документтерин көзөмөлдөө жана башкаруу үчүн толук стек веб-колдонмо. Бардык интерфейс кыргыз тилинде.

## Технологиялар

- **Frontend:** Next.js 15, React 19, TypeScript, TailwindCSS
- **Backend:** Next.js API Routes
- **База:** SQLite (Prisma ORM) — жергиликтүү интранет орнотуу үчүн
- **Экспорт:** xlsx
- **Графиктер:** Recharts

## Орнотуу

### Талаптар

- Node.js 18+ 
- npm

### Кадамдар

```bash
# 1. Тийиндиктерди орнотуу
npm install

# 2. Базаны түзүү жана толтуруу
npm run db:setup

# 3. Ишке киргизүү
npm run dev
```

Браузерде ачыңыз: [http://localhost:3000](http://localhost:3000)

## Тест колдонуучулары

Бардык колдонуучулардын сырсөзү: `123456`

| Логин | Роль |
|-------|------|
| `archive_staff` | Архив кызматкери |
| `reading_room` | Окуу залынын кызматкери |
| `head_dept` | Бөлүм башчысы |
| `vault_staff` | Сактоочу жайдын кызматкери |
| `management` | Жогорку жетекчилик |

## Ролдор жана функциялар

### 1. Архив кызматкери
- Ички муктаждык үчүн билдирме түзүү
- Жеке тарыхты көрүү

### 2. Окуу залынын кызматкери
- Изилдөөчүлөрдү каттоо (Санариптик профиль)
- Кагаз документ суроолорун каттоо
- Excel экспорт

### 3. Бөлүм башчысы
- Күтүүдөгү билдирмелерди макулдоо/четке кагуу
- Статистика көрүнүшү
- Деңгээл 1 эскертүүлөр (бузулган документтер)

### 4. Сактоочу жайдын кызматкери
- Макулдаштырылган билдирмелерди иштетүү
- 2-кадамдык текшерүү (логин/сырсөз кайра киргизүү)
- Кайтаруу учурунда сапат текшерүүсү (Текшерүү)
- Микроклиматты көзөмөлдөө

### 5. Жогорку жетекчилик
- Окуу гана аналитикалык панель
- Графиктер (статус, максат, микроклимат)
- Excel экспорт
- Өзгөртүүлөрдүн тарыхы (Логирование)

## Бизнес-логика

### Инцидент эскалациясы
- Сапат текшерүүсү ийгиликсиз болсо → автоматтык Акт (деңгээл 1)
- 24 саат ичинде чечилбесе → деңгээл 2 (Жогорку жетекчиликке)

### Микроклимат эскертүүсү
- T > 21°C же H > 60% → "КРИТИКАЛЫК АБАЛ! Темир шарттар бузулду!"

### Аудит лог
- Бардык статус өзгөртүүлөр катталат
- Жогорку жетекчилик үчүн окулуу гана

## API маршруттары

| Маршрут | Метод | Сыпаттама |
|---------|-------|-----------|
| `/api/auth/login` | POST/DELETE | Кирүү/чыгуу |
| `/api/auth/me` | GET | Учурдагы колдонуучу |
| `/api/auth/verify` | POST | 2-кадамдык текшерүү |
| `/api/requests` | GET/POST | Билдирмелер |
| `/api/requests/[id]` | PATCH | Статус өзгөртүү |
| `/api/quality-checks` | POST | Сапат текшерүүсү |
| `/api/climate` | GET/POST | Микроклимат |
| `/api/alerts` | GET/PATCH | Эскертүүлөр |
| `/api/audit-logs` | GET | Аудит лог |
| `/api/documents` | GET | Документтер |
| `/api/researchers` | GET/POST | Изилдөөчүлөр |
| `/api/statistics` | GET | Статистика |
